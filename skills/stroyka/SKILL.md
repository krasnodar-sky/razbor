# Стройка — фазовый конвейер создания

Дисциплина против главного источника переделок: неверно понятых требований и первого попавшегося решения. Модель сама никогда не переспрашивает (берёт вероятную трактовку и уверенно едет) и не выбирает между вариантами (продолжает самый вероятный токен — второй вариант просто не существует в разговоре). Конвейер заставляет её сначала понять, потом сравнить, и лишь потом строить.

Сигнал масштаба: от ~получаса работы или больше пары файлов/сущностей → полный цикл. Мелочь — без церемонии. Сломалось/не работает → конвейер razbor, не этот.

## 1. Понять (BRIEF)
Шаблон: `${CLAUDE_PLUGIN_ROOT}/templates/BRIEF.md`. Заполняется В ДИАЛОГЕ: черновик пишет агент, вопросы задаёт владельцу — иначе это бюрократия в одиночестве.
- Формулировка СВОИМИ словами: копия запроса ничего не проверяет, пересказ проверяет понимание.
- Критерии успеха — только проверяемые. «Работает лучше» — не критерий; «повторный клик не создаёт второго письма» — критерий. Нет ни одного проверяемого критерия → задача непонятна, строить РАНО.
- Открытые вопросы закрывай проверкой (история изменений, журнал, воспроизведение), не догадкой. Догадка называется «предполагаю».
- Решение не предлагается, пока критерии не сформулированы.

## 2. Спланировать (PLAN)
Шаблон: `${CLAUDE_PLUGIN_ROOT}/templates/PLAN.md`. Пишется ДО работы: пока не вложено ничего, смена пути стоит ноль; модель, написавшая 200 строк первого варианта, будет его защищать.
- Шаги проверяемые: про каждый можно ответить «сделан / не сделан». «Разобраться с X» — не шаг; «найти все точки записи X и составить список» — шаг.
- ОБЯЗАТЕЛЬНО ≥1 отвергнутая альтернатива с причиной, называющей ЦЕНУ: «дольше», «шире скоуп», «не устраняет причину», «добавляет зависимость». «Отвергнута, потому что плохо» — ритуальная отписка, переделать. Приём: «предложи два подхода, сравни, обоснуй выбор и причину отказа».
- Риски светофором: 🔴 красный решается ДО начала работы, не после.
- План утверждает человек. Реализация до утверждения не начинается.

## Турникет между фазами
Не переходи к следующей фазе, пока не показал человеку чек-лист выхода текущей (последняя секция шаблона): процитируй пункты, отметь ✅/❌, при ❌ — скажи чего не хватает. Пункт нельзя отметить ✅ без содержимого в секции. «Чек-лист пройден ✅» без цитаты пунктов — то же проскакивание, только с галочкой. Отказ идти дальше при провале — успех механизма, а не сбой.

## 3. Сделать
По шагам плана. Отклонение от плана называй до, а не после. Многоэтапное — механизм этапов с проверкой на каждом шаге (у fablize — goals.py).

## 4. Закрепить
Как фиксация в razbor: итог с причиной → журнал системы; новая грабля → реестр граблей; недоделки/обещания → персистентный носитель, не чат. Черновики записей предлагает агент, человек утверждает — 5 минут, которые делают систему умнее с каждой задачей.
