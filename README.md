# razbor — харнес-конвейер разбора проблем

Дисциплина решения проблем для ИИ-агентов: интейк с проверкой на рецидив → критерий «решено» → диагностика (3+ гипотезы, улики, отвергнутые в отчёт) → решение → независимая верификация → фиксация. Плюс живой чек-лист типовых граблей ([packs/grabli.md](packs/grabli.md)).

Родился не из теории: паттерны добыты майнингом реальных журналов работы ИИ-агентов (2 месяца инцидентов), драфт прошёл адверсарное ревью, обкатан на боевых разборах. Чек-лист граблей пополняется из практики.

## Установка

**Claude Code** (плагин):
```bash
claude plugin marketplace add krasnodar-sky/razbor
claude plugin install razbor@razbor
```
Скилл `razbor` триггерится фразами «разбери проблему», «найди дыру», «в чём корень». Чтобы конвейер включался на любой входящей проблеме автоматически — добавь строку роутинга в CLAUDE.md своей машины (пример ниже).

**ChatGPT / другой ассистент:** скопируй [adapters/chatgpt.md](adapters/chatgpt.md) в Custom Instructions / Custom GPT / Project.

**Голый markdown:** ядро самодостаточно — скорми `skills/razbor/SKILL.md` + `packs/grabli.md` как системный текст любому агенту.

## Строка роутинга для CLAUDE.md (опционально)

```
- **[входящая проблема или рецидив: «не работает», «найди дыру», «в чём корень», «что сломалось»]**
  Веди по конвейеру скилла razbor: интейк с проверкой на рецидив → критерий «решено» → диагностика →
  решение → независимая верификация → фиксация. Не для тривиальных вопросов-фактов.
```

## Второй конвейер: stroyka (стройка)

`razbor` чинит сломанное, `stroyka` строит новое. Скилл `stroyka` триггерится на нетривиальную задачу-создание («сделай», «построй», «запусти», «внедри») и ведёт её фазами: **Понять** (BRIEF: формулировка своими словами + проверяемые критерии успеха — нет критерия → строить рано) → **Спланировать** (PLAN: проверяемые шаги + ≥1 отвергнутая альтернатива с причиной-ценой + риски-светофор с «необратимое = 🔴 по умолчанию» + бюджет с правилом стопа) → репетиция для рискованных планов (прогон с конкретикой + пре-мортем) → турникет (чек-лист выхода с цитированием пунктов) → **Сделать** (превысил бюджет → стоп и развилка, не молча) → **Закрепить** (сделано / отклонения / сюрпризы). Мелочь — малым режимом, без артефактов. Шаблоны: [templates/BRIEF.md](templates/BRIEF.md), [templates/PLAN.md](templates/PLAN.md).

Раз в квартал — аудит харнеса по [packs/audit.md](packs/audit.md): четыре метрики «до → после», вердикты правилам (живое/мёртвое/хромое) и минимум одно вычеркивание — цифры вместо «вроде помогает».

Работаете с несколькими LLM (Claude + другие)? [packs/multimodel.md](packs/multimodel.md) — протокол «один харнес, много моделей»: общие правила и память, тег модели в записях, ритуалы входа-выхода для сред без хуков, подключение новой модели чтением.

Строка роутинга:

```
- **[крупная задача-стройка: «сделай», «построй», «запусти», «внедри» — от ~получаса или больше пары файлов]**
  Веди по конвейеру скилла stroyka: BRIEF (критерии успеха до работы) → PLAN (отвергнутая альтернатива с ценой) →
  турникет с цитированием чек-листа → Сделать → Закрепить. Мелочь — без церемонии; починка — конвейер razbor.
```

## Обновление (боевой цикл)

Новая грабля в разборе → обобщи до паттерна «сигнал → проверка» → допиши в `packs/grabli.md` → commit + push → на машинах:
```bash
claude plugin update razbor
```
Локальную конкретику (пути, каналы, ID своих систем) — в память/CLAUDE.md своей машины, не в плагин.

## Философия

- Харнес не поднимает потолок модели — он доводит её до её собственного потолка процедурой.
- Минимальный роутинг: применяй только ту дисциплину, на которую есть сигнал задачи.
- Честная эскалация: застрял 2+ раза — назови предел, не изображай прогресс.
- Дружит с [fablize](https://github.com/fivetaku/fablize): если тот установлен, диагностика использует его investigation-protocol pack, многоэтапность — goals.py.
