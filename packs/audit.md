# Аудит харнеса — цифры вместо «вроде помогает»

Раз в квартал (или когда возник вопрос «а работает ли это всё»). «Вроде стало лучше» — не аргумент: не для тебя через месяц, не для того, кому порекомендуешь. Без измерения не видно, какая часть системы несёт нагрузку, а какая — мёртвый груз.

## 1. Боли: было → сейчас → что изменило
Вернись к списку болей, с которых харнес начинался (первые записи журнала/граблей). По каждой три ответа: как было / как сейчас / какой компонент системы это изменил. Компонент, который не всплыл ни в одной боли и ни в одной метрике, — первый кандидат на упрощение.

## 2. Четыре метрики (грубые цифры, точность не нужна)
| Метрика | Как считать |
|---|---|
| Переделки с нуля | сколько раз результат выброшен целиком (журнал/память) |
| Дубли существующего | агент предложил то, что уже есть в системе |
| Время ввода в контекст | минуты от нового чата до содержательной работы |
| Повторные ошибки | одна и та же ошибка дважды ПРИ записи в реестре граблей |

«Было 2 в неделю → стало 2 в месяц» — достаточная строгость. Таблица «до → после» = ответ на «помогает ли» и точка отсчёта следующего аудита.

## 3. Аудит правил: живое / мёртвое / хромое
Пройди правила харнеса (грабли-пак, роутинг-строки, конституцию) построчно, каждому — вердикт:
- **живое** — видел, как сработало (агент остановился на бюджете, нашёл рецидив, процитировал чек-лист) → остаётся;
- **мёртвое** — за отчётный период ни разу не повлияло на поведение → ВЫЧЕРКНУТЬ, не «пусть лежит»: мёртвые правила разбавляют живые, короткий живой список сильнее длинного мёртвого. Минимум одно вычеркивание за аудит;
- **хромое** — должно работать, но игнорируется → переформулировать короче и императивнее, вторая попытка до следующего аудита.

## 4. Кастомизация
Шаблоны универсальные — твоя работа нет. Структуру фаз не трогаем (Понять → Спланировать → Сделать → Закрепить работает везде), наполнение секций подгоняй под тип работы: для кода критерии через тесты и бюджет файлами; для текстов — аудитория и действия читателя, бюджет разделами; для анализа — источники и проверяемость выводов.

## Выход аудита
Запись в журнал системы: что вычеркнуто, что переформулировано, что кастомизировано, таблица метрик. Метка следующего аудита (+3 месяца) в персистентный носитель.
