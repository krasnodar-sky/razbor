# Changelog

## 1.3.0 — 2026-07-23
- **Новый pack packs/multimodel.md — «Один харнес, много моделей»:** харнес принадлежит системе, не модели; одна память/журнал на всех с тегом модели в записях; ритуалы входа-выхода сессии обязательны для любой модели (автоматика-хуки = усилитель, не условие — логика в обычных скриптах, зовущихся из любой среды); подключение новой модели чтением, не пересказом; перенос харнеса ≠ перенос полировки (явный слой — файлами, рефлекс — повторами и обкаткой).
- Повод: в системе появился второй LLM-двигатель, работающий по тем же правилам той же памятью.

## 1.2.0 — 2026-07-18
- **stroyka дозрел до полного ядра курса (уроки 8–13):** правило «необратимое = 🔴 по умолчанию, подтверждение у границы необратимости — даже из утверждённого плана»; процедура бюджета (превысил → стоп → цифры → развилка «новый план / откат», молчаливое «ещё чуть-чуть» запрещено); секция «Репетиция» (прогон плана с конкретикой, метки «НЕ УВЕРЕН», пре-мортем — для внешних систем/необратимого/цепочек); фаза «Закрепить» со структурой «сделано / отклонения / сюрпризы»; малый режим с правилами эскалации (растёт → BRIEF задним числом; сюрприз мелочи → всё равно в грабли).
- **Новый pack packs/audit.md** — квартальный аудит харнеса: 4 метрики «до → после», вердикты правилам (живое/мёртвое/хромое, минимум одно вычеркивание), боли «было → сейчас → что изменило».
- templates/PLAN.md: калибровка рисков, правило стопа в бюджете, секция репетиции.
- Источник: уроки 8–13 «Harness Starter Kit» (транскрипты + авторские RULES v2.1/v2.2, PLAN v2.2).
